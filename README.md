# Shimeji Desktop Pet

스프라이트 시트 PNG 한 장으로 나만의 Windows 데스크톱 펫을 만들어 주는 템플릿입니다.  
캐릭터는 바탕화면을 걸어 다니며, 마우스로 잡아서 옮기거나 크기를 변경할 수 있습니다.

> 제작자: jjinzxx  
> 제작일: 2026-06-25  
> 마지막 업데이트: 2026-07-22  
> GitHub: [Shimeji-Desktop-Pet](https://github.com/jjinzxx/Shimeji-Desktop-Pet)

## 지원 환경

- Windows 10 또는 Windows 11
- 인터넷 연결: 처음 EXE를 만들 때 필요
- Python: 미리 설치하지 않아도 됨

## 다운로드

1. 이 GitHub 페이지 위쪽의 초록색 **Code** 버튼을 누릅니다.
2. **Download ZIP**을 선택합니다.
3. 내려받은 ZIP 파일의 압축을 풉니다.

Git을 사용할 수 있다면 다음 명령으로 받을 수도 있습니다.

```bash
git clone https://github.com/jjinzxx/Shimeji-Desktop-Pet.git
```

## 나만의 데스크톱 펫 만들기

1. 사용할 스프라이트 시트 PNG 한 장을 준비합니다.
2. PNG를 압축 해제한 폴더의 `build.bat`과 같은 위치에 넣습니다.
3. 폴더 바로 아래에 PNG가 정확히 한 장만 있는지 확인합니다.
4. `build.bat`을 더블클릭합니다.
5. 만들 EXE의 이름을 입력합니다. 이름을 입력하지 않고 Enter를 누르면 `ShimejiPet`이 사용됩니다.
6. 빌드가 끝나면 `dist` 폴더 안에 완성된 EXE가 생성됩니다.

Python이 설치되어 있지 않으면 BAT가 자동으로 설치합니다. 이후 Pillow와 PyInstaller를 설치하고 EXE를 만들기 때문에 첫 빌드는 시간이 조금 걸릴 수 있습니다. 완성된 EXE를 실행하는 컴퓨터에는 Python이 필요하지 않습니다.

선택 사항으로 `icon.ico`를 `build.bat` 옆에 넣으면 완성된 EXE의 아이콘으로 사용됩니다.

## 스프라이트 시트 규격

전체 이미지는 같은 크기의 **4열 × 4행** 격자여야 합니다. 각 프레임의 크기는 이미지 전체 크기에서 자동으로 계산됩니다.

| 행 | 캐릭터 동작 | 왼쪽부터 사용하는 칸 |
|---|---|---:|
| 1행 | 가만히 있기 | 2칸 |
| 2행 | 걷기 | 4칸 |
| 3행 | 떨어지기 | 2칸 |
| 4행 | 잡혀 있기 | 1칸 |

예를 들어 전체 이미지가 128×160px이면 각 프레임은 자동으로 32×40px이 됩니다. 사용하지 않는 칸은 투명하게 비워 두어도 됩니다. 캐릭터 바깥 배경이 투명한 PNG를 권장합니다.

격자 배치는 [예제 스프라이트 시트](example/example_sheet.png)를 참고하세요. 예제 폴더의 이미지는 EXE를 만들 때 포함되지 않습니다.

### 상태별 프레임 수 변경

기본 설정은 다음과 같습니다.

```python
# sprite.py
STATE_FRAMES = {
    "idle": 2,     # 가만히 있을 때 2프레임
    "walk": 4,     # 걸을 때 4프레임
    "fall": 2,     # 떨어질 때 2프레임
    "grabbed": 1,  # 마우스로 잡혀 있을 때 1프레임
}
```

상태별 그림 수가 다르면 [sprite.py](sprite.py)의 `STATE_FRAMES` 숫자를 바꾸면 됩니다. 예를 들어 걷기 그림을 3장만 사용할 경우 다음처럼 수정합니다.

```python
"walk": 3,
```

각 숫자는 해당 행에서 **왼쪽부터 실제로 사용할 프레임 수**입니다. 기본 시트는 가로 4칸이므로 1~4 사이로 설정해야 하며, 숫자만큼의 그림을 빈칸 없이 왼쪽부터 배치해야 합니다.

상태가 배치된 행은 같은 파일의 `STATE_ROWS`에서 정합니다. 행 번호는 0부터 시작합니다.

```python
# sprite.py
STATE_ROWS = {
    "idle": 0,     # 첫 번째 행
    "walk": 1,     # 두 번째 행
    "fall": 2,     # 세 번째 행
    "grabbed": 3,  # 네 번째 행
}
```

격자 자체를 4×4가 아닌 형태로 만들고 싶다면 `GRID_COLS`와 `GRID_ROWS`도 바꿀 수 있습니다. 이 경우 모든 `STATE_ROWS` 값은 `0` 이상 `GRID_ROWS - 1` 이하여야 하고, 모든 `STATE_FRAMES` 값은 `1` 이상 `GRID_COLS` 이하여야 합니다.

## 세부 설정 변경

[config.py](config.py)에서 크기, 애니메이션 속도, 걷는 속도, 중력과 행동 지속 시간을 조정할 수 있습니다. 값을 바꾼 뒤 `build.bat`을 다시 실행해야 새 EXE에 반영됩니다.

| 설정 | 기본값 | 설명 |
|---|---:|---|
| `SCALE` | `5` | 원본 프레임을 화면에 표시할 배율입니다. `5`이면 가로·세로가 각각 5배가 됩니다. |
| `TRANSPARENT_COLOR` | `"#010101"` | Windows가 창 배경으로 인식해 투명하게 처리할 색입니다. 캐릭터 그림에 이 색을 사용하면 그 부분도 투명해질 수 있습니다. |
| `TICK_MS` | `16` | 움직임을 갱신하는 간격(ms)입니다. 작을수록 더 자주 갱신됩니다. 보통 `16`을 유지하는 것을 권장합니다. |
| `ANIM_TICK` | `12` | 한 애니메이션 프레임을 유지하는 업데이트 횟수입니다. 작을수록 애니메이션이 빨라집니다. |
| `WALK_SPEED` | `1.0` | 한 번 업데이트할 때 가로로 이동하는 화면 픽셀 수입니다. 클수록 빨리 걷습니다. |
| `GRAVITY` | `0.6` | 떨어질 때 매 업데이트마다 추가되는 낙하 속도입니다. 클수록 빠르게 가속됩니다. |
| `MAX_FALL_SPEED` | `14.0` | 낙하 속도의 최댓값입니다. 너무 빠르게 떨어지는 것을 막습니다. |
| `IDLE_DURATION` | `(60, 200)` | 가만히 있는 시간의 최소·최대 업데이트 횟수입니다. 범위 안에서 매번 무작위로 정해집니다. |
| `WALK_DURATION` | `(80, 200)` | 한 번 걷는 시간의 최소·최대 업데이트 횟수입니다. 범위 안에서 매번 무작위로 정해집니다. |

예를 들어 캐릭터를 기본보다 2배 빠르게 걷게 하고 애니메이션도 빠르게 만들려면 다음처럼 바꿀 수 있습니다.

```python
# config.py
ANIM_TICK: int = 6
WALK_SPEED: float = 2.0
```

`CHAR_W`, `CHAR_H`, `DISP_W`, `DISP_H`는 스프라이트 시트와 배율을 기준으로 자동 계산되므로 직접 수정할 필요가 없습니다.

## 사용 방법

완성된 `dist\이름.exe`를 더블클릭하면 데스크톱 펫이 실행됩니다.

| 조작 | 동작 |
|---|---|
| 마우스 왼쪽 버튼으로 드래그 | 캐릭터를 잡아서 이동 |
| 마우스 오른쪽 버튼 클릭 | 크기 설정 및 종료 메뉴 열기 |

## 문제가 생겼을 때

- **PNG가 없다는 메시지:** PNG를 `build.bat`과 같은 폴더에 넣으세요.
- **PNG가 여러 장이라는 메시지:** 폴더 바로 아래에는 사용할 PNG 한 장만 남기세요. `example` 폴더의 이미지는 상관없습니다.
- **4×4로 나누어지지 않는다는 메시지:** 이미지의 가로와 세로 픽셀 수가 각각 4로 정확히 나누어져야 합니다.
- **Python을 설치하지 못했다는 메시지:** 인터넷 연결을 확인한 뒤 BAT를 다시 실행하세요.
- **Windows 보안 경고:** 직접 만든 서명되지 않은 EXE이므로 SmartScreen 경고가 나타날 수 있습니다.
